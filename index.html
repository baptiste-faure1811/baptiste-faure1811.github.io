Privacy

At AppyCare, we strongly believe that privacy is a core principle that should be considered a human right. With that in mind, we take your privacy (and your users'), extremely seriously. We are committed to being transparent about what information we collect and the resons why we might collect any specific data.

Our BDPM API does not collect any usage data from you or your users. We do not track how you use this API or what you do with the information you receive through it. We value your trust and are committed to offer you an easy access to medication data without making any compromises about your privacy.

If you have any questions or concerns about your privacy, or any related questions, please do not hesitate to contact us at privacy@appycare.fr.

Headers

A few HTTP headers are included in every response messages that are sent between you and our servers. These headers provide additional information about the request or response, such as the content type, the date and time, and general information about the API.

These HTTP headers can be used to retrieve informations such as how to contact our support team or get the official data source link.

Global API key

This is a consumption-only API, only the HTTP GET method is available no matter the ressource or endpoint used. No individual authentication is required to access this API, and resources are fully open and available using this global API key:

8c7d3f5f-c9b1-419a-b93f-cafe2f7c41c4

To be granted access to the API ressources, the API key needs to be passed using a HTTP header.

Use the AC-API-KEY header name to pass the global key; note that any request made that do not include a valid key will return a 401 Unauthorized error. No usage rate applies, but we still encourage you to limit the frequency of requests to limit our hosting costs and more importently limit data transfers to the stricly required ones.

Example using ExpressJS :
req.set("AC-API-KEY", "lsdqkjfnlkjndlfqkjbkjb");

Please be aware that the value of this global key might change in the future and would require you to update your requests to use the updated value.
